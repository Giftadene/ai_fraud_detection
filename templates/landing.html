auto_mode Automated Setup (Recommended)

  1. Download the project ZIP using the button below.
  2. Extract the ZIP to any folder (e.g., C:\FraudGuardAI).
  3. Double-click start.bat — this will:
    • Verify Python is installed
    • Create a virtual environment
    • Install Flask, Pandas, NumPy, Scikit-learn, and XGBoost
    • Train the ML model
    • Launch the web server at http://127.0.0.1:5000
    • Open your browser to the home page
  4. Click "Sign In" in the top navigation bar and log in with the credentials above.

terminal Manual Command Line Setup

Open a terminal in the project folder and run:

python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
set FRAUDGUARD_ADMIN_PW=admin123
set FRAUDGUARD_ANALYST_PW=analyst123
python app.py

Then open http://127.0.0.1:5000 in your browser.

package_2 Building a Standalone Executable (No Python Required)

For distribution to users who do not have Python installed, you can build a single-file executable using PyInstaller:

  1. Install PyInstaller: pip install pyinstaller
  2. Run the build script: build_exe.bat
  3. Find the executable in the dist\FraudGuardAI folder
  4. Share the executable — no Python or libraries needed on the target machine
info

The executable bundles Python, Flask, XGBoost, and all dependencies into a single portable package. Users simply double-click to run.

Frequently Asked Questions

Common questions about the FraudGuard AI system.

What is FraudGuard AI?expand_more

FraudGuard AI is a machine learning-based fraud detection system for online banking and mobile money transactions. It uses XGBoost algorithms to analyze transaction behavior in real-time and predict whether a transaction is legitimate or fraudulent.

Do I need to know programming to use it?expand_more

No. The start.bat script handles everything automatically. If you use the pre-built executable version, you don't even need Python installed. Just double-click and the system runs.

What machine learning model does it use?expand_more

The system uses XGBoost (Extreme Gradient Boosting), which is the industry standard for fraud detection. It achieves over 95% accuracy on synthetic transaction data. The model can be retrained from the Model Lab interface at any time.

Can I use real transaction data?expand_more

Yes. The system is designed to work with real banking data. The synthetic data included is for demonstration and testing. You can replace the data source by modifying the database and retraining the model through the Model Lab.

Is the system secure?expand_more

The system uses Flask sessions with signed cookies, role-based access control, password hashing with SHA-256, and environment variable configuration for credentials. For production deployment, additional measures like HTTPS encryption and database encryption are recommended.

What are the system requirements?expand_more

Minimum 4GB RAM, 500MB disk space, and a modern web browser. The system runs on Windows, macOS, and Linux. Python 3.10+ is required for the source code version; the pre-built executable requires no dependencies.

How do I reset admin credentials?expand_more

Set the FRAUDGUARD_ADMIN_PW and FRAUDGUARD_ANALYST_PW environment variables before running the app. If the database is deleted and recreated, these values will be used as the default passwords.
