Product Requirements Document (PRD)

Documentation chevron_right System Specifications

Machine Learning-Based Fraud Detection System Specification

1. Project Overview

Detect fraudulent attempts on online banking and mobile money channels in real-time, utilizing feature engineering and classifier modeling (XGBoost / RandomForest) to calculate decision weights and route requests.

2. Scoring Classification Routing

Score RangeAction Path
0 - 30Approve Immediately
31 - 60Monitor Activity
61 - 80Hold for Review (Queue Alert)
81 - 100Block Transaction

3. Evaluated Model Features

  • amount_ratio: Transaction Amount vs average spending baseline.
  • location_distance: Distance (KM) shift from usual locations.
  • hour_of_day: Night transactions flag (unusual velocity shift).
  • device_changed: Device type fingerprint signature mismatch.
  • vpn_usage: Encrypted networks routing proxy tunnel detection.
  • velocity: Request velocity threshold spikes.
  • previous_fraud: Historical flags check.
